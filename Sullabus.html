1 INFORMATIONS GÉNÉRALES

Code module B3PRJ2 - Hackathon Applications Sociales Cloud-Native
Niveau Bachelor 3 - Unité 2
Durée 5 jours / 35 heures
Format Équipes de 3 à 4 étudiants maximum
Prérequis Docker avancé, Kubernetes, CI/CD GitLab, développement full-stack

2 PRÉREQUIS DÉTAILLÉS

Acquis requis de B3U1

  • Automatisation système : Bash, scripts d'infrastructure
  • Infrastructure as Code : Ansible, Terraform
  • DevOps culture : Principes d'automatisation et collaboration
  • Git avancé : Stratégies de branches, workflows collaboratifs

Acquis requis de B3U2

  • Docker avancé : Optimisation d'images, réseaux, sécurité
  • Kubernetes : Architecture, composants, configuration
  • CI/CD GitLab : Pipelines avancés, déploiement automatisé
  • Gestion de projet Agile : Méthodes collaboratives

Compétences développement requises

  • Frontend : React, Vue.js ou technologies équivalentes
  • Backend : Python (Django/Flask), Node.js ou équivalent
  • Bases de données : Relationnelles et NoSQL
  • APIs : REST, intégration services tiers

3 OBJECTIFS PÉDAGOGIQUES

À la fin de ce hackathon, l'étudiant sera capable de :

  • Concevoir et développer une application sociale moderne en architecture cloud-native
  • Intégrer harmonieusement containerisation Docker et orchestration Kubernetes
  • Implémenter des pipelines CI/CD complets avec GitLab
  • Développer des fonctionnalités temps réel et des interfaces utilisateur engageantes
  • Gérer un projet complexe en équipe avec contraintes temporelles strictes
  • Appliquer les bonnes pratiques DevOps en situation de production
  • Présenter et défendre ses choix techniques devant un jury

4 SUJETS DE PROJET

Chaque équipe doit choisir un seul sujet parmi les 5 propositions suivantes. Tous les projets doivent être développés en architecture cloud-native avec containerisation Docker obligatoire.

Option D - Discord-Gaming Plus : Plateforme Social Gaming

Concept : Plateforme de communication avec mini-jeux en browser intégrés, tournois inter-campus et fonctionnalités gaming sociales.

Fonctionnalités attendues :

  • Communication : Chat vocal/texte temps réel
  • Mini-jeux : Jeux browser simples intégrés (Pong, Snake, Trivia)
  • Tournois : Compétitions officielles inter-campus
  • Communautés : Guildes par école/formation
  • Partage : Spectateur mode pour regarder ses amis jouer

Stack technique suggérée : Vue.js, Node.js, Socket.io, WebRTC

5 CONTRAINTES TECHNIQUES OBLIGATOIRES

⚠️ Contraintes non-négociables
  • Architecture containerisée : Tous les services doivent être containerisés avec Docker
  • Multi-services : Minimum 3 conteneurs (frontend, backend, base de données)
  • Pipeline CI/CD : GitLab CI configuré avec déploiement automatisé
  • Interface utilisateur : Interface moderne et responsive
  • Fonctionnement complet : Application déployée et accessible

Exigences de qualité

  • Code source : Repository Git propre avec commits fréquents
  • Documentation : README détaillé avec instructions de déploiement
  • Sécurité : Authentification utilisateur implémentée
  • Performance : Temps de réponse acceptables
  • Données : Persistance et intégrité assurées
🎯 BONUS KUBERNETES (+20% note finale)

Orchestration Kubernetes : Les équipes qui déploient leur application sur Kubernetes avec configuration complète (services, ingress, volumes) bénéficieront d'un bonus de 20% sur leur note finale.

Critères bonus :

  • Déploiement fonctionnel sur cluster Kubernetes
  • Configuration des services et load balancing
  • Gestion des volumes persistants pour les données
  • Configuration ingress pour accès externe

6 DÉROULEMENT ET MÉTHODOLOGIE

Planning indicatif

  • Jour 1 matin : Formation des équipes, choix du sujet, architecture
  • Jour 1 après-midi : Setup initial, containerisation de base
  • Jours 2-3 : Développement des fonctionnalités core
  • Jour 4 : Intégration, tests, amélioration UX
  • Jour 5 matin : Finalisation, préparation des démos
  • Jour 5 après-midi : Présentations devant jury

Méthodes de travail attendues

  • Autonomie : Recherche de solutions, débrouillardise technique
  • Collaboration : Répartition des tâches, entraide dans l'équipe
  • Itération : Développement incrémental avec démonstrations régulières
  • Pragmatisme : Priorisation des fonctionnalités essentielles
  • Innovation : Créativité dans les solutions techniques

7 ÉVALUATION

Critères d'évaluation (note sur 20)

Technique (40%)

  • Architecture containerisée complète
  • Qualité du code et bonnes pratiques
  • Fonctionnement de l'application
  • Pipeline CI/CD opérationnel

Innovation & UX (30%)

  • Originalité des fonctionnalités
  • Interface utilisateur attrayante
  • Expérience utilisateur fluide
  • Créativité dans la résolution

Collaboration (20%)

  • Répartition équilibrée des tâches
  • Entraide et communication
  • Gestion du repository Git
  • Documentation collaborative

Présentation (10%)

  • Démonstration en direct
  • Explication des choix techniques
  • Défense du projet
  • Gestion du temps de présentation
Bonus Kubernetes : +20% de la note finale si déploiement K8s complet

Modalités de présentation

  • Durée : 15 minutes par équipe (10 min demo + 5 min questions)
  • Format : Démonstration en direct de l'application
  • Livrable : URL de l'application déployée + repository Git
  • Documentation : README complet avec guide d'installation

8 RESSOURCES ET OUTILS

Infrastructure mise à disposition

  • GitLab : Repositories privés avec CI/CD runners
  • Environnement cloud : Machines virtuelles pour déploiement
  • Cluster Kubernetes : Accès pour équipes bonus
  • Bases de données : Instances PostgreSQL, MongoDB

Technologies recommandées

  • Frontend : React, Vue.js, React Native pour mobile
  • Backend : Node.js, Python (Django/Flask), ou équivalent
  • Bases de données : PostgreSQL, MongoDB, Redis
  • 3D/Gaming : Three.js, Canvas API, Socket.io
  • APIs tiers : Google Maps, GitHub, LinkedIn, etc.

Support technique

  • Encadrement : Formateurs disponibles pour déblocage
  • Documentation : Accès aux ressources de cours
  • Pair programming : Encouragé entre équipes
  • Recherche autonome : Internet, documentation officielle
